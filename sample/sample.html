sugarskull has a very flexible routing table.

The routing table is a JSON object that connects full or partial routes with functions.

The keys in the JSON object that start with a forward slash are considered routes. Routes can be complete or broken up into parts and nested. They can be nested as deeply as you would like.

They may contain regular expressions or string replacement `:tokens` anywhere you like. 
var routerInstance = Router({ ... });
a constructor that can be passed a simple object literal.

routerInstance.init();
once initialized sugarskull will start monitoring the state of the document hash, links that change it will affect the router. 
routerInstance.use({ ... })
accepts an object literal of global events, such as functions to get fired on, after or once for every route in the routing table. 
routerInstance.getRoute([index])
get the current route. if an index is provided the route will be treated as an array (delimited by the `/` character), the index of the route will then be returned. 
routerInstance.setRoute(...)
set the current route to a string provided as a parameter. if a numeric index and string value are provided the route will be treated as an array (delimited by the `/` character), the index of the route will then be set to the string provided. if two numeric values are provided sections of the route will be removed starting from parameter 1 and ending at parameter 2.

sometimes you only need a wrench, not the toolbox with a hammer, screwdriver, etc. sugarskull is small. it does one job and does it well.

It's not a framework, its a simple tool easy to add or remove. It promotes centralizing router logic so it's not intertwined throughout your code.

Sugarskull was intended to replace backbone.js routes and provide a lighter weight, less sinatra-like alternative to sammy.js.
